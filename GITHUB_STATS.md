# GitHub Stats Setup

The homepage reads `data/github-stats.json` for the GitHub summary card. A GitHub Actions workflow can update that file every day without exposing any token to browser JavaScript.

## Private-inclusive contribution setup

Create a fine-grained personal access token and store it as the repository secret `GH_STATS_TOKEN`.

Recommended token settings:

- Resource owner: your GitHub account
- Repository access: only repositories you want included
- Permissions: read-only repository metadata and contents

The workflow intentionally uses this secret instead of the default `GITHUB_TOKEN`, because the default token cannot count private commits across your account. Browser JavaScript also cannot query private commits safely, so the homepage displays `-- commits / 30d` until `data/github-stats.json` is generated by the workflow.

Then go to:

1. GitHub repository
2. Settings
3. Secrets and variables
4. Actions
5. New repository secret
6. Name: `GH_STATS_TOKEN`
7. Value: your token

## Run it

Go to:

1. Actions
2. Update GitHub Stats
3. Run workflow

The workflow writes `data/github-stats.json` and commits it if the value changed. It also runs daily at `03:17 UTC`.

## Change username

The GitHub username is set in two places:

- `script.js`: `githubUsername`
- `.github/workflows/update-github-stats.yml`: `GITHUB_STATS_USERNAME`

Both are currently set to `barrydoooit`.
